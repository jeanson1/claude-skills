# Exemples avant/après — contexte IA et communication pro

Les exemples couvrent plusieurs cas d'usage : hook éditorial, article, post LinkedIn, email professionnel, description de produit. Les sujets cités (Claude, Claude Code, outils dev) servent de support narratif sans orienter le skill vers un domaine unique.

---

## Exemple 1 — Hook éditorial : throat-clearing + négation-contraste + tournure pompeuse

**Avant (slop) :**
> "Voici ce que personne ne te dit sur Claude Code. Ce n'est pas juste un assistant, c'est un véritable partenaire qui vient bouleverser ta façon de coder. Et ça change tout."

**Après (sans-fioritures) :**
> "Claude Code lit ton code, modifie 12 fichiers d'un coup et lance les tests tout seul. Tu vas comprendre pourquoi les juniors le maîtrisent mieux que les seniors."

**Changements** : suppression "voici ce que personne ne te dit" (throat-clearing + hyperbole guru), suppression négation-contraste, suppression "véritable partenaire" + "bouleverser" + "change tout" (vague), remplacement par actions concrètes + thèse spécifique.

---

## Exemple 2 — Description Substack : tics journalistiques + emphase creuse

**Avant (slop) :**
> "À l'heure où l'IA générative redéfinit les contours du métier de développeur, force est de constater que les outils comme Claude Code marquent un véritable tournant. Il n'en demeure pas moins que beaucoup peinent à les adopter."

**Après (sans-fioritures) :**
> "Claude Code change ton métier de développeur cette année. Tu peines à l'adopter ? Voici la méthode qui m'a pris 3 semaines."

**Changements** : "à l'heure où" supprimé, "redéfinit les contours" → action directe, "force est de constater" supprimé, "marque un tournant" → "change ton métier" + "cette année" (spécifique), "il n'en demeure pas moins" → "tu peines à l'adopter ?", ajout d'engagement direct.

---

## Exemple 3 — Post LinkedIn : false agency + filler + restatement

**Avant (slop) :**
> "L'IA bouleverse les codes du support client. Les algorithmes décident désormais quels tickets passent en priorité. Concrètement, ça veut dire que ton message n'est plus lu par un humain. Autrement dit, tu écris pour une machine."

**Après (sans-fioritures) :**
> "Anthropic a sorti un modèle qui trie les tickets support avant l'agent humain. Ton message passe d'abord par une IA. Écris pour les deux : balises claires pour la machine, ton direct pour l'agent."

**Changements** : "l'IA bouleverse" → "Anthropic a sorti X" (acteur nommé), "les algorithmes décident" → action concrète, "concrètement ça veut dire" supprimé, "autrement dit" supprimé, conclusion actionnable au lieu de constat.

---

## Exemple 4 — Email pro : jargon professionnel + nous éditorial

**Avant (slop) :**
> "Nous avons constaté que l'embarquement des équipes sur Claude Code nécessite une approche structurée. Nous proposons d'adresser ce sujet via un workshop pour driver la transformation et identifier les quick wins."

**Après (sans-fioritures) :**
> "Tes équipes adoptent mal Claude Code. Je propose un workshop de 2h cette semaine : on lance 3 cas d'usage concrets et on choisit celui qui marche le mieux."

**Changements** : "nous avons constaté" → "tes équipes adoptent mal" (direct), "embarquement", "adresser", "driver", "transformation", "quick wins" tous supprimés, ajout temporel concret + livrable mesurable.

---

## Exemple 5 — Article Substack long-form : tournures pompeuses + narrateur distant

**Avant (slop) :**
> "On observe que les développeurs qui se révèlent les plus performants en 2026 sont ceux qui ont su s'approprier les outils d'IA. Cette nouvelle génération vient s'inscrire dans une lignée de praticiens qui questionnent les certitudes du métier."

**Après (sans-fioritures) :**
> "Les développeurs les plus rapides en 2026 utilisent Claude Code 4 heures par jour. Ils ont remplacé Stack Overflow et la moitié des code reviews. Tu peux faire pareil avec 2 semaines d'apprentissage."

**Changements** : "on observe que" supprimé, "se révèlent" → mesure spécifique (4h/jour), "vient s'inscrire dans une lignée" supprimé, "questionnent les certitudes" → action spécifique (remplacé Stack Overflow), conclusion actionnable.

## Exemple 6 — Hook court (mode social) : fragmentation dramatique + hyperbole

**Avant (slop) :**
> "Le secret que personne ne te dit. Claude Code. C'est tout. C'est ça le futur du dev. Plot twist : tu peux commencer aujourd'hui."

**Après (sans-fioritures, mode social) :**
> "Claude Code remplace 60% de tes recherches Stack Overflow.
> Mesure faite sur mes 3 derniers projets.
> Tu peux tester en 10 minutes."

**Changements** : "le secret que personne ne te dit" supprimé (anti-guru), fragmentation dramatique supprimée, "plot twist" supprimé, ajout chiffre + preuve + action (10 min).

---

## Exemple 7 — Mode minuscules (Substack intime / hot take)

**Avant (slop) :**
> "Voici ce que j'ai compris en utilisant Claude Code pendant 6 mois. C'est tout simplement le meilleur outil dev de la décennie. Honnêtement, je n'ai jamais vu une telle accélération de productivité."

**Après (sans-fioritures, mode minuscules) :**
> "6 mois sur claude code et je code 3x plus vite sur mes projets perso. pas sur le travail d'équipe (la review humaine ralentit). l'écart est sur le solo : prototypes, scripts, side-projects."

**Changements** : "voici ce que j'ai compris" supprimé, "c'est tout simplement le meilleur" → mesure (3x), "honnêtement" supprimé, "jamais vu" → spécification (perso vs équipe), capitalisation supprimée pour ton intime.

## Exemple 8 — Description de contenu SEO : closings + menus conditionnels

**Avant (slop) :**
> "Dans cette vidéo, on va voir ensemble comment utiliser Claude Code. En conclusion, c'est un outil incroyable. J'espère que cette vidéo t'aide. Si tu veux, je peux faire une vidéo plus avancée. N'hésite pas à me dire en commentaire."

**Après (sans-fioritures) :**
> "Cette vidéo te montre comment utiliser Claude Code en 12 minutes : setup, premier projet, et 3 erreurs à éviter en mode agent.
>
> Au moment du tournage : Claude Code 2.4 sur macOS.
>
> Outils mentionnés :
> - Claude Code (claude.ai/code)
> - VS Code
>
> Mes prédictions sur l'évolution agentique : voir l'épisode du 12 mars."

**Changements** : "on va voir ensemble" → "cette vidéo te montre" (direct), "en conclusion" supprimé, "incroyable" supprimé, "j'espère que ça t'aide" supprimé, menu conditionnel supprimé, ajout structure SEO concrète.

## Exemple 9 — Réponse à un commentaire : hedging + reformulation

**Avant (slop) :**
> "Je pense que c'est une bonne question. À mon humble avis, il me semble que Claude Code est plus adapté pour le solo. Concrètement, ça veut dire qu'en équipe il y a peut-être des frictions. En d'autres termes, le bénéfice est asymétrique."

**Après (sans-fioritures) :**
> "Claude Code donne un avantage en solo et frictionne en équipe. La review humaine ralentit le cycle. En solo, tu prototypes 3x plus vite. En équipe, attends-toi à un gain de 20-30% max."

**Changements** : "je pense que" / "à mon humble avis" / "il me semble" supprimés, "concrètement ça veut dire" supprimé, "en d'autres termes" supprimé, ajout chiffres + condition spécifique.

## Exemple 10 — Newsletter intro : hyperbole guru + tics LinkedIn

**Avant (slop) :**
> "Spoiler alert : tout ce que tu sais sur l'IA est faux. Hot take : les outils comme Claude Code vont rendre 80% des devs juniors obsolètes. Petit rappel : tu lis ça sur Substack, pas sur LinkedIn."

**Après (sans-fioritures) :**
> "Les juniors qui maîtrisent Claude Code remplacent les seniors qui ne s'y mettent pas. J'ai vu 3 cas dans des équipes que j'accompagne en mars. Voici la liste des compétences à monter en priorité, par ordre d'impact."

**Changements** : "spoiler alert" supprimé, "tout ce que tu sais est faux" supprimé, "hot take" supprimé, "petit rappel" supprimé, ajout preuve (3 cas observés en mars) + livrable annoncé.

---

## Exemple 11 — Texte long : voix passive + faux pluriel + adverbes

**Avant (slop) :**
> "Il est généralement admis que les outils d'IA générative ont fondamentalement transformé le développement logiciel. Nous pensons sincèrement que cette tendance va s'accélérer. Personnellement, nous observons que beaucoup d'équipes peinent à réellement intégrer ces outils."

**Après (sans-fioritures) :**
> "Les outils d'IA changent le développement depuis 18 mois. La tendance va s'accélérer en 2026. J'observe que les équipes peinent à intégrer ces outils, surtout sur la review de code."

**Changements** : "il est admis que" → direct (voix active), "fondamentalement", "sincèrement", "personnellement", "réellement" tous supprimés, "nous" → "j'" (assumé), ajout spécificité (review de code).

---

## Exemple 12 — Punchline finale : restatement + tournure pompeuse

**Avant (slop) :**
> "Pour résumer, ce que je veux te dire, c'est que l'IA n'est pas une menace, mais un levier. Autrement dit, ce qui change la donne, c'est ta capacité à t'approprier ces outils. En clair : commence aujourd'hui."

**Après (sans-fioritures) :**
> "Commence aujourd'hui. Choisis un projet de la semaine prochaine. Fais-le avec Claude Code. Mesure le gain. Recommence."

**Changements** : "pour résumer" supprimé, négation-contraste supprimée, "ce qui change la donne" supprimé, "autrement dit" supprimé, "en clair" supprimé, remplacement par 5 actions concrètes courtes.

## Pattern récurrent : la conversion "oral guru" → "écrit direct"

| Pattern oral guru | Réécriture sans-fioritures |
|---|---|
| "Imagine que tu pouvais X" | "Avec X, tu peux Y" |
| "Et si je te disais que..." | "Voilà ce qui se passe : ..." (ou direct) |
| "Tu vas voir, c'est dingue" | "Voici la mesure : ..." |
| "Le truc qu'il faut comprendre" | (couper le préambule, énoncer) |
| "Ça va te changer la vie" | "Tu gagnes X minutes par Y" |
| "Tout le monde devrait..." | "Si tu fais X, alors Y. Sinon, Z." |
| "C'est révolutionnaire" | "C'est nouveau parce que [spécificité]" |

## Test final : la phrase passe-t-elle ?

Avant de livrer, pose-toi ces 3 questions :

1. **Cette phrase peut-elle s'appliquer à n'importe quel autre sujet ?** Si oui, elle est vide.
2. **Y a-t-il un humain qui fait quelque chose dans cette phrase ?** Si non, refais.
3. **Si je supprime cette phrase, le texte perd-il quelque chose ?** Si non, supprime.
